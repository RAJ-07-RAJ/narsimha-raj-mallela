PRJ-001  ·  Processor Design

RISC-V RV32I Single-Cycle Processor

Verilog RTL  ·  Artix-7 XC7A35T  ·  Vivado 2023.2
+
Overview

Full implementation of a synthesizable single-cycle RISC-V RV32I processor supporting R-type, I-type, and load/store instructions. The control path is fully combinational; datapath is 32-bit wide throughout. Low device utilization is expected — single-cycle design with a small instruction subset and simple ALU datapath.

RTL — Control Decoder (excerpt)
module control_unit (
  input  logic [6:0] opcode,
  input  logic [2:0] funct3,
  input  logic [6:0] funct7,
  output logic        reg_write, mem_read, mem_write,
  output logic        alu_src, mem_to_reg, branch,
  output logic [1:0] alu_op
);
  // RISC-V RV32I combinational decode — opcode → control signals
  always_comb begin
    unique case (opcode)
      7'b0110011: begin  // R-type (add, sub, and, or, xor...)
        reg_write=1; alu_src=0; mem_to_reg=0;
        mem_read=0; mem_write=0; branch=0;
        alu_op=2'b10;
      end
      7'b0000011: begin  // I-type load (lw)
        reg_write=1; alu_src=1; mem_to_reg=1;
        mem_read=1; mem_write=0; branch=0;
        alu_op=2'b00;
      end
      7'b0100011: begin  // S-type store (sw)
        reg_write=0; alu_src=1; mem_to_reg=0;
        mem_read=0; mem_write=1; branch=0;
        alu_op=2'b00;
      end
      7'b1100011: begin  // B-type branch (beq, bne)
        reg_write=0; alu_src=0; mem_to_reg=0;
        mem_read=0; mem_write=0; branch=1;
        alu_op=2'b01;
      end
      // ... addi (7'b0010011), jal (7'b1101111) ...
    endcase
  end
endmodule

  • I-type: addi, andi, ori, xori, slti, slli, srli
  • Load / Store: lw, sw
  • Branch: beq, bne, blt, bge
  • Jump: jal, jalr

PRJ-002  ·  CDC Design

Asynchronous FIFO — Dual Clock Domain Crossing

Verilog / SystemVerilog  ·  Gray-coded pointers  ·  2-FF synchronizers  ·  SV Assertions
+
The CDC Problem

When data crosses between two unrelated clock domains, metastability is a real failure mode — not a theoretical one. A synchronizer with insufficient stages can propagate a metastable output into downstream logic and corrupt state. This FIFO solves CDC by never passing multi-bit binary pointers directly across domains.

Design Strategy — Gray Coding

Binary pointers are converted to Gray code before synchronization. Gray code guarantees that between any two adjacent pointer values, only one bit changes — meaning a 2-FF synchronizer on each bit can settle before being sampled, with no intermediate illegal states possible.

// Binary → Gray conversion
assign wptr_gray = wptr_bin ^ (wptr_bin >> 1);
assign rptr_gray = rptr_bin ^ (rptr_bin >> 1);

// 2-FF synchronizer — read pointer into write domain
always_ff @(posedge wclk) begin
  rptr_sync1 <= rptr_gray;
  rptr_sync2 <= rptr_sync1;  // rptr_sync2 is metastability-safe
end

// SystemVerilog assertion — no overflow
assert property (@(posedge wclk) disable iff (!wrst_n)
  !(winc && wfull)) else $error("FIFO overflow detected");
RTL — Parameterized Interface
module async_fifo #(
  parameter DATA_WIDTH = 8,
  parameter DEPTH      = 16,   // must be power-of-2
  parameter ADDR_WIDTH = $clog2(DEPTH)
) (
  input  logic                  wclk, wrst_n,
  input  logic                  rclk, rrst_n,
  input  logic                  winc, rinc,
  input  logic [DATA_WIDTH-1:0] wdata,
  output logic [DATA_WIDTH-1:0] rdata,
  output logic                  wfull, rempty
);
Architecture — Modules
fifo_mem (dual-port) wptr_handler rptr_handler sync_r2w (2-FF) sync_w2r (2-FF) full_logic empty_logic
Full / Empty Flag Logic

Full is asserted in the write domain: synchronized read pointer matches write pointer with MSB inverted — indicating write has lapped read. Empty is asserted in the read domain when both pointers are equal. The extra pointer bit (ADDR_WIDTH+1 bits total) eliminates full/empty ambiguity entirely.

CDC Safety Features
  • Gray-coded read/write pointers — only 1 bit changes between adjacent values
  • Two-stage flip-flop synchronizers on both crossing directions (r2w and w2r)
  • Full/empty detection fully contained within respective clock domains
  • SystemVerilog assertions for overflow and underflow detection at simulation time

PRJ-003  ·  High-Speed Interface

AXI4-Stream Master–Slave Pipeline

TVALID/TREADY handshake  ·  Backpressure  ·  TLAST packet boundaries  ·  Skid buffer
+
Overview

AXI4-Stream master and slave with a registered pipeline stage between them. The design correctly propagates backpressure, handles packet boundaries using TLAST, and achieves full throughput (one transfer per cycle) under continuous TREADY assertion.

Handshake Protocol

A transfer occurs only when both TVALID (master-driven) and TREADY (slave-driven) are asserted simultaneously. Per the AXI4-Stream spec, the master must not deassert TVALID once raised unless the transfer has completed. Violating this causes silent pipeline stalls that are difficult to debug in simulation.

RTL — Master Interface
module axis_master #(
  parameter DATA_WIDTH = 32
) (
  input  logic                  clk, rst_n,
  output logic [DATA_WIDTH-1:0] m_axis_tdata,
  output logic                  m_axis_tvalid,
  output logic                  m_axis_tlast,
  input  logic                  m_axis_tready
);
  // Transfer completes only when both handshake signals high
  assign transfer = m_axis_tvalid & m_axis_tready;
Backpressure — Skid Buffer

The pipeline stage between master and slave uses a skid buffer. When the slave deasserts TREADY, the in-flight transfer is captured in a side register. The master sees TREADY deasserted one cycle later (registered), giving it one extra cycle to hold valid data — preventing loss without requiring the master to re-drive stale data.
